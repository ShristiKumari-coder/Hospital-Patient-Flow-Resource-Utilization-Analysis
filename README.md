# Hospital-Patient-Flow-Resource-Utilization-Analysis
Hospital Operational Analytics project analyzing patient encounter data to identify peak arrival times, queue congestion, department utilization, and seasonal demand patterns. Using Python, SQL, and visualization, it provides insights to optimize staffing, improve hospital efficiency, and reduce patient waiting time by ~15% during peak hours.

Project Overview-

Hospitals often face challenges such as:
Overcrowding during peak hours.
Long patient waiting times.
Uneven staff workload.
Poor resource allocation.

This project uses healthcare data analytics to understand hospital operations and provide actionable recommendations for improving efficiency.


Business Objectives-

The main objectives of this project are:
Identify peak hours and peak days of patient arrivals.
Analyze queue congestion patterns.
Evaluate department utilization and workload.
Detect monthly seasonal trends in hospital visits.
Create a staffing plan to reduce patient waiting time by 15%.

Analytics Performed-

1️. Exploratory Data Analysis (EDA)
Understanding the structure of hospital encounter data and identifying patterns.

2️. Peak Hour Analysis
Detecting the busiest hours and days in hospital operations.

3️. Queue Analysis
Analyzing hourly patient queue sizes to identify congestion periods.

4️. Department Utilization
Evaluating patient visits and treatment durations across departments.

5️. Time Series Analysis
Studying daily patient arrival trends using moving averages.

6️. Seasonality Analysis
Identifying monthly patterns in hospital demand.


Key Insights-

Certain hours experience very high patient arrivals.
Some departments receive significantly more visits than others.
Patient queues increase significantly during peak hours.
Hospital visits show seasonal patterns across months.
Adding ~20 additional staff during peak hours can help reduce waiting time by ~15%.
